## Caveats

- **Duplicated rows:** logs recorded before the 2026-07-03 tick-loop fix (`dd83717`) that crossed continuous-runs rollovers contain duplicated rows (up to 3/sec). Mirrors intentionally replay rows **exactly as recorded** — both engines see the identical sequence, so the head-to-head is fair — but per-tick counts from those bars overweight them; dedupe by second if per-bar weighting matters in a future analysis.
- The comparator's console output carries legacy row labels ("v5.1/v5.3") from the template it was forked from; in `v5.4/analysis/replay-compare.mjs` OLD = v5.3 and NEW = v5.4. Cosmetic only.
- Mirrors are counterfactuals. The standing rule stands: **never treat `_v54m` (or `_v53m`) as live sessions.**
